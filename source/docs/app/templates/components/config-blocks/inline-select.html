Configuration:

Property Values Explanation
ng-model Angular expression The model representing the selected element
placeholder String Text displayed when the select's text field is empty or the select is opened
oncommit Function

Function to be called when the selected value value is committed

Parameters:

  • $data contains the entered value to be committed

Return value:

  • If the function returns a promise, the inline text field will display a spinning wheel until the promise is resolved or rejected.
  • If the promise gets resolved, a green success indicator is displayed.
  • If the promise gets rejected, a red error indicator is displayed. Additionally, the parameter with which the promise was rejected will be displayed as the error message underneath the text field.
  • If the function does not return a promise, the green success indicator is shown immediately.

items Array The array of values to be used as the select's possible values
tabindex Number The index of this form field for tab navigation
disabled Angular expression If this expression evaluates to true, the input field will be disabled.
display-property String If the values in the items array are objects, this argument can be used to determine which property of those objects will be used as the display value of the selected element when the select is closed.
position String In which direction the dropdown should open. See UI Select documentation
refresh Function

Function used to dynamically load source data for this select's items.

Parameters:

  • $query The search string typed into the select's text field by the user

See UI Select documentation

refresh-delay Number See UI Select documentation
allow-clear Angular expression If the expression evaluates to true, the content of the inline-select will be clearable via a button.
allow-commit-unchanged Angular expression If the expression evaluates to true, the oncommit handler will be called regardless of whether the selected value actually changed or not.

The HTML inside the data-inline-select element will be used as a template for the select items when the select is opened. The current item object is available in the item variable. Apply the highlight filter as demonstrated above to visually highlight the search text entered by the user inside the select items. If no child element of data-inline-select is supplied, the following HTML will be used as the default item template: 

<span data-ng-bind-html="item | highlight:$select.search"></span>
