Configuration:

Property Values Explanation
notification-title string (optional) The string representing the popover title. If no title is given, the title header within the popover will not be displayed.
notification-template string Representing the location of a template to use for the popover body. Note that the contents of this template need to be wrapped in a tag, e.g.: <div></div>
notification-data object

The data object contains all notification data. The data object has following required properties:

count, highlightedNotifications, highlightedNotificationsTitle, notifications, notificationsTitle
data.count number This number will displayed in the red bubble and shows the number of new notifications and/or highlighted notifications.
data.highlightedNotificationsTitle string (optional) Optional title for highlighted notifications section. If no title is given the section title will not be displayed.
data.highlightedNotifications array

Array including highlightedNotification objects. If the array is empty the highlighted notifications section will not be displayed. The highlightedNotifications object has following required properties:

icon, handled, username, action, message, date

You can add more properties (e.g. id), if the notification functions (e.g. handleUrl or markAllAsRead) need them.
data.notificationsTitle string (optional) Optional title for notifications section. If no title is given the section title will not be displayed.
data.notifications array

Array including notification objects. If the array is empty the notifications section will not be displayed. The notifications object has following required properties:

unread, icon, username, action, message, date

You can add more properties (e.g. id), if the notification functions (e.g. handleUrl or markAllAsRead) need them.
data.notifications[i].handled boolean A boolean flag to show the handled (read) status of an notification.
data.notifications[i].username string A string to display the name of the notification triggering person.
data.notifications[i].action string A short text about the action.
data.notifications[i].message string (optional) A additional (more detailed) message for an action.
data.notifications[i].date Date The notification date.
data.notifications[i].icon object

Icon object to display an icon for the notification. The icon object has following required properties:

symbol, color
data.notifications[i].icon.symbol string A font awesome icon e.g. fa-check.
data.notifications[i].icon.color string A color class.
notification-load-more function (optional) A custom function that will be called if the infinite scroll event ist triggered. This function should update the data object.
notification-handle function (optional) A custom function that will be called if a notification is clicked (redirect to an specific page, mark notification as read, ...). If no function is available a click will not do anything. The function (as directive property) should always have '{notification: notification}' as function argument. If a notification is clicked, the directive will return a notification object that contains all the data of the clicked notification.
notification-handle-highlighted function (optional) A custom function that will be called if a highlighted notification is clicked (redirect to an specific page, mark highlighted notification as read, ...). If no function is available a click will not do anything. The function (as directive property) should always have '{notification: notification}' as function argument. If a notification is clicked, the directive will return a notification object that contains all the data of the clicked notification.
notification-go-to-overview-page function (optional) A custom function that will be called if the 'See all' link is clicked (redirect to an specific page). If no function is available the see all link will not be displayed.
notification-mark-all-as-read function (optional) A custom function with additional behavior will be called if the 'mark all as read' link is clicked. In this case the (directive) internal function will set the handled flag of all notifications (except the highlighted notifications) to true
