Inline Edits

For detail pages and other areas where data should be viewable and quickly editable at the same time, there are directives to achieve that for various types of input.

They all work on the same principle that as soon as the input element loses focus, the oncommit callback function is called. If this callback function returns a promise, a spinning wheel is displayed until the promise is resolved or rejected, at which point success or error feedback is shown where the spinner previously was. The parameter given to the reject function will be used as the error message and displayed unterneath the input element. If the callback function does not return a promise, success feedback is shown instantly.

Forms using inline edits on detail pages should arrange them in a two column layout like suggested below and on the detail page.

Inline text input

<div class="inline-form">
    <div class="row">
        <div class="col-xs-12">
            <div class="form-group">
                <div class="col-xs-4 col-sm-2 col-md-2 label-container">
                    <label for="user-firstname-text">First name</label>
                </div>
                <div class="col-xs-8 col-sm-4 col-md-3">
                    <data-inline-text
                        id="user-firstname-text"
                        data-ng-model="inlineCtrl.data.user.firstname"
                        placeholder="First name"
                        oncommit="inlineCtrl.sendToServer($data)"></data-inline-text>
                </div>
                <div class="clearfix visible-xs"></div>
                <div class="narrow col-md-2 hidden-xs hidden-sm"></div>

                <div class="col-xs-4 col-sm-2 col-md-2 label-container">
                    <label for="user-surname-text">Surname</label>
                </div>
                <div class="col-xs-8 col-sm-4 col-md-3">
                    <data-inline-text
                        id="user-surname-text"
                        data-ng-model="inlineCtrl.data.user.lastname"
                        placeholder="Surname"
                        oncommit="inlineCtrl.saveImmediately($data)"></data-inline-text>
                </div>
                <div class="clearfix visible-xs"></div>
            </div>
        </div>
    </div>
</div>

Inline datepicker

The datepicker supports all datepickerOptions that the UI Bootstrap datepicker supports. 

<inline-datepicker
    id="user-birthdate-picker"
    data-ng-model="inlineCtrl.data.user.birthDate"
    datepicker-options="inlineCtrl.birthDatepickerOptions"
    placeholder="Pick your birth date"
    date-format="dd/MM/yyyy"
    popup-placement="auto top bottom"
    oncommit="inlineCtrl.sendToServer($data)"></inline-datepicker>

Inline typeahead

The typeahead is based on the UI Bootstrap typeahead component. 

<data-inline-typeahead
    id="state-of-birth-typeahead"
    data-ng-model="inlineCtrl.data.user.stateOfBirth"
    placeholder="Search states …"
    items="inlineCtrl.states"
    oncommit="inlineCtrl.sendToServer($data)"></data-inline-typeahead>

Inline select

The select field is based on the UI Select directive. The HTML inside the data-inline-select element will be used as a template for the select items when the select is opened. The current item object is available in the item variable. Apply the highlight filter as demonstrated below to visually highlight the search text entered by the user inside the select items. The attribute display-property determines which property of the item objects is used as the display value when the select is closed.

<data-inline-select
    id="site-select"
    data-ng-model="inlineCtrl.data.user.favoriteArtist"
    display-property="name"
    placeholder="Select or search an artist in the list …"
    items="inlineCtrl.artists"
    oncommit="inlineCtrl.sendToServer($data)">
    <div>
        <span data-ng-bind-html="(item.name + ' (' + item.age + ')') | highlight:$select.search"></span><br />
        <small>Country: <span data-ng-bind-html="item.country | highlight:$select.search"></span></small>
    </div>
</data-inline-select>
